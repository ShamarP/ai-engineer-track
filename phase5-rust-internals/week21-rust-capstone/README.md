## 🛠 Build — pick **A**, **B**, or **C**

### A — Speculative decoding in your engine
- [ ] Add spec decoding to your Week 20 inference engine
- [ ] Use a small draft model (e.g., Llama-3-1B drafting for Llama-3-8B)
- [ ] Measure acceptance rate and speedup
- [ ] Read [Leviathan et al., 2023, *Speculative Decoding*](https://arxiv.org/abs/2211.17192), sections 1–3

### B — A real PR
- [ ] Pick `vllm`, `candle`, `mistral.rs`, or `llama.cpp`
- [ ] Browse `good-first-issue` labels
- [ ] Fix one. Open the PR. Even a docs/perf fix counts.
- [ ] Read the project's `CONTRIBUTING.md` and trace the maintainer review workflow

### C — Open-source your engine
- [ ] Polish your Week 20 mini inference engine: docs, examples, README, GitHub Actions CI, MIT license, name
- [ ] Submit to Hacker News (`Show HN: A 1500-line Rust inference engine I wrote in 4 weeks`)
- [ ] Read open-source-launch playbooks (e.g., supabase, plausible launch posts)

## ✅ Done when
- [ ] **(A)** Spec decoding shows measurable speedup on at least one prompt class; benchmarks in README
- [ ] **(B)** PR is open with at least one maintainer comment
- [ ] **(C)** Repo is public with a clear README; submitted to HN or Lobste.rs; first external star from a stranger

## 🎯 Stretch
- [ ] Whichever path you didn't pick — do a second one. They compound. Your engine + a real vLLM PR is a *very* strong portfolio.
